The 1.1 release is backwards-compatible but we recommend you upgrade your libraries so you can take advantage of the improved APIs, new features and functionality, and fixes and improvements.

New features and functionality
Transient Publishing
Publish to a channel without attaching to it and no longer worry about the client inadvertently receiving all messages published on that channel.
Push Notifications
Thanks to those of you who got involved in beta testing push notifications. After a some tweaking push notifications are now generally available!
Batch REST publishing
The experimental batch REST publishing API allows a single REST operation to publish one or more messages to one or more channels in a single operation.
Access to the DSX
Some client libraries now support access to the new Ably Data Stream Exchange (DSX) stats. If you try the DSX, we'd love to hear what you think about it.
.NET and go love
We massively upgraded the .NET and go libraries from 0.8 to 1.1. Yep, we skipped a 1.0 release.
iOS goes multi-platform
iOS is now multi-platform with support for tvOS and macOS. We're excited about this one.

Dive into the docs
We've updated our client library SDK docs to reflect changes in 1.1.
